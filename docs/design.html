Glossary Design System

A strict-monochrome e-ink design system for Glossary, a distraction-free Wikipedia lookup and flashcard PWA, derived from Kindle e-reader reference imagery.

Human-readable mirror of docs/design.md (the source of truth for coding agents). Same tokens, rendered live — toggle the theme top right. The only hue on any page is the highlight yellow.

Colors

Strict monochrome: an ink ramp on e-ink paper, plus one yellow reserved for reader-saved highlights. Every text/background pair meets WCAG AA in both themes. No accent, no red — interaction is contrast; safety is confirmation.

Typography

Literata (serif) = everything read. Mulish (sans) = everything operated. All sizes in rem: the font-size setting changes one root variable and the whole app scales.

Spacing

0.25rem-base scale; rem units mean rhythm scales with the reader's font-size preference. Touch targets never dip below 44px.

Radius

Pill = type-and-tap immediacy (search, chips, thumbs). sm = buttons, toolbar, dialogs — the near-square device feel. md = resting containers. Nothing in between.

Elevation & Depth

Borders and contrast first, like the panel the design imitates. Two shadow levels, always paired with a hairline border. Motion: ≤150ms fades, none under reduced-motion.

Account & export

login/account page (#/login) · three states in the device voice · signed out: quiet pitch + one outline button + Privacy/Terms text links · signing in: progress · signed in: grayscale avatar, name, email, Sign out, and Delete account as the quiet-destructive pattern (types-DELETE confirm, never red)
first-login import · a device already holding local cards gets a ONE-TIME offer to import them — never a silent merge (a shared machine must not leak one person's cards into another's account) · the same import lives in Settings → Sync afterwards
export sheet · signed in adds EMAIL TO ME beside Download · bundles the selection as "(DD-MM-YYYY) Glossary Export.zip" · phone: straight to the share sheet · computer: zip downloads while a pre-addressed Gmail compose tab opens — a caption on the sheet says exactly that, so the manual attach step is never a surprise

Do's and Don'ts

✓ Do

  • Reference every color, size, and radius through its token — no raw hex/px in component CSS outside :root.
  • Keep the serif/sans split absolute: Literata is read, Mulish is operated.
  • Size and space in rem so the font-size setting scales the whole interface coherently.
  • Give every interactive element a visible 2px ink focus outline — keyboard is a first-class path.
  • Define light and dark values together whenever a color token is added.
  • Express hierarchy with contrast, borders, and weight first; shadows only at the two defined levels.

✗ Don't

  • Don't introduce any accent hue — yellow means "saved by the reader" and nothing else carries color.
  • Don't add a third font family, a third surface level, or a radius between md and full.
  • Don't style destructive actions red — quiet text button plus confirmation, like the device.
  • Don't animate for attention: ≤150ms fades only, none under reduced motion.
  • Don't render anything that looks like an outbound link — nothing here navigates away.
  • Don't let chrome grow: if a screen needs explanatory text, reconsider the screen.
