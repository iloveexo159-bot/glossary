# Decisions

- 2026-07-16 Mobile top-bar chrome (settings + account) defaults to COLLAPSED on pages with the top search — over default-expanded because the whole point of the toggle is a full-width search bar; both icons stay one tap away behind «.
- 2026-07-16 Flip sound ships as a same-origin file (`app/audio/flip-card.mp3`) played via a lazily-created `Audio` element, with `'self'` added to the CSP `media-src` — over Web Audio fetch/decode (more code for no gain) and a base64 data URI (bloats app.js ~41KB and needs `media-src data:`).
- 2026-07-16 The Collection page head (title, card search, toolbar, selection bar, filters panel) wraps in ONE sticky `.cards-head` on ≤640px — over keeping the selection bar's separate sticky rule, because two sticky elements with the same `top` offset overlap each other during scroll.
- 2026-07-16 Card-flip duration derives from the flip audio's own metadata at runtime (`durationchange` → `--flip-duration` CSS var, 250ms fallback) — over hardcoding the clip length, so editing the mp3 retimes the animation with no code change; `durationchange` over `loadedmetadata` because an mp3's duration is an estimate the browser refines as it decodes.
- 2026-07-17 Datamuse `/sug` chosen as the typo-correction source for search suggestions — over Wikipedia opensearch `profile=fuzzy` and `list=search&srinfo=suggestion` because both were probed live and fail on the motivating typo ("viviseting" → ViVi junk / empty result), while Datamuse returns "vivisecting" as its top hit; Datamuse is already a dependency (`ml=` synonyms) and already in the CSP connect-src.
- 2026-07-17 Tapping the "Did you mean" correction row submits immediately (definition-first via `submitDefine`) — over rewriting the search box for a second confirming tap, because it matches how every other dropdown row behaves and the corrected spelling still appears prominently as the result-page headword; chosen by the user from two mocked-up flows.
- 2026-07-16 Flip sound REMOVED same day (supersedes the two entries above) — the reader disliked the sound. The whole surface reverted: audio asset, playback code, `media-src 'self'`, SW precache entry, and the audio-synced `--flip-duration` var; the flip is a fixed 250ms again. `design-system/flip-card-sound-track.mp3` (the source asset) is kept.
